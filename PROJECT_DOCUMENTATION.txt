# Tanishq Luxury Jewelry E-Commerce Platform
## Full Project Documentation

### Project Overview
Tanishq is a comprehensive luxury jewelry e-commerce platform built for a hackathon. It features AI-powered product recommendations, personalized user experiences, admin dashboard, and a complete shopping workflow.

### Tech Stack
- **Backend**: Python 3.13, FastAPI, SQLAlchemy, SQLite
- **Frontend**: Vanilla HTML/CSS/JavaScript (no frameworks)
- **AI**: OpenAI SDK with Titan AI Gateway integration
- **Authentication**: JWT tokens, bcrypt password hashing
- **SMS**: Fast2SMS API for phone verification
- **Deployment**: Single-server architecture (FastAPI serves both API and static files)

### Key Features

#### User Features
- **Product Catalog**: Browse jewelry by categories (Rings, Necklaces, Earrings, Bracelets, Pendants, Bangles)
- **AI Chatbot**: Conversational jewelry consultant powered by Titan AI Gateway
- **Personalized Recommendations**: AI-driven product suggestions based on user preferences
- **User Authentication**: Email/password login with OTP verification
- **Phone Verification**: SMS OTP for enhanced security
- **Profile Management**: Complete user profiles with DOB/anniversary tracking
- **Shopping Cart & Wishlist**: Full e-commerce functionality
- **Guest Checkout**: Shop without registration
- **Order Management**: Complete order lifecycle
- **Promo Codes**: Discount system

#### Admin Features
- **Dashboard**: Comprehensive admin panel with analytics
- **User Management**: View all users and their activity
- **Session Analytics**: Chat session monitoring
- **Recommendation Tracking**: AI recommendation performance
- **Product Management**: Full CRUD operations (via database)

#### AI Features
- **Intelligent Chatbot**: Context-aware conversations using Titan AI Gateway
- **Preference Learning**: Tracks user browsing patterns
- **Personalized Categories**: Reorders category grid based on user interests
- **Smart Recommendations**: AI-powered product suggestions
- **Conversation History**: Persistent chat sessions

### Database Schema

#### Users Table
- id (Primary Key)
- name, email, password_hash
- role (user/admin)
- email_verified, phone, phone_verified
- date_of_birth, anniversary_date
- created_at, last_seen, segment

#### Items Table (Products)
- id (Primary Key)
- name, category, subcategory
- description, price, original_price
- image_url, rating, reviews_count
- in_stock, featured, attributes (JSON)

#### Sessions Table (Chat Sessions)
- id (Primary Key)
- user_id (Foreign Key)
- started_at, path, message_count

#### Messages Table (Chat History)
- id (Primary Key)
- session_id (Foreign Key)
- role (user/assistant), content, timestamp
- intent, constraints (JSON), item_ids_shown

#### Preferences Table (User Preferences)
- id (Primary Key)
- user_id (Foreign Key, Unique)
- categories, styles, past_likes, past_dislikes (JSON)

#### Cart/Wishlist/Order Tables
- Standard e-commerce relationships with foreign keys

### AI Integration

#### Titan AI Gateway
- Base URL: https://ai.titan.in/gateway
- Model: azure/gpt-5-nano
- Authentication: JWT Bearer token
- Fallback: Direct OpenAI API

#### Chatbot Flow
1. Router Agent: Determines conversation path (preference/new user)
2. Need-State Agent: Gathers user requirements
3. Preference Agent: Retrieves personalized recommendations
4. Recommendation Agent: Finds matching products
5. OpenAI Service: Generates natural responses

### Security Features
- API Key Middleware: Protects all /api/ routes
- JWT Authentication: Stateless token-based auth
- Password Hashing: bcrypt with salt
- CORS Configuration: Controlled cross-origin access
- Input Validation: Pydantic models
- SQL Injection Prevention: SQLAlchemy ORM

### Performance Optimizations
- Lazy Loading: Images load on demand
- Session Storage: Popup states persist
- Local Storage: Auth tokens and guest IDs
- Database Indexing: Optimized queries
- Static File Caching: Browser caching headers

### Deployment Options
- **Development**: Local FastAPI server

- **Quick Demo**: ngrok tunneling for public access
